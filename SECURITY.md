# Security Policy

## Supported versions

Only the latest commit on the default branch is considered supported.

## Reporting vulnerabilities

Please report vulnerabilities privately and do not open a public issue.

Include:

- Affected component (`mobile`, `desktop`, protocol, discovery, pairing)
- Reproduction steps
- Impact assessment
- Suggested mitigation (if available)

Until a private reporting mailbox is configured, contact project maintainers directly.

## Disclosure policy

1. We acknowledge reports quickly.
2. We triage and reproduce before public disclosure.
3. We ship a fix and publish advisory notes after remediation.
