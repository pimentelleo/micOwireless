# Code signing policy

Free code signing provided by [SignPath.io](https://about.signpath.io), certificate by [SignPath Foundation](https://signpath.org).

## Scope

This policy applies to released binaries distributed by this repository, especially Windows `.exe` and `.msi` artifacts.

## Team roles

- Committers and reviewers: [@pimentelleo](https://github.com/pimentelleo) and project contributors listed in [Contributors](https://github.com/pimentelleo/micOwireless/graphs/contributors).
- Approvers: [@pimentelleo](https://github.com/pimentelleo).

## Signing rules

1. Sign only binaries produced from this repository.
2. Sign only binaries generated by auditable CI workflows from tagged releases.
3. Do not sign unrelated third-party applications or binaries from non-project repositories.
4. Keep product metadata consistent across signed artifacts.
5. Keep release artifacts uninstallable and documented.

## Security requirements

1. Maintainers and approvers must keep MFA enabled on GitHub and on signing systems.
2. Signing credentials must be stored in repository secrets, never in source code.
3. Any suspected signing compromise requires immediate credential rotation and release revocation.
